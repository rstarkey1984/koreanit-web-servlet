## 1. 웹(Web)이란?

- 웹(Web)은 전세계의 **네트워크**가 연결된 **인터넷** 위에서 HTTP/HTTPS 프로토콜을 사용해 클라이언트와 서버가 정보를 주고받는 시스템입니다. 우리가 사용하는 웹사이트나 API 서비스 모두 이 방식으로 작동합니다

  “인터넷”은 물리적으로 연결된 초거대 네트워크 연결망이고, “웹”은 그 위에서 동작하는 서비스입니다.

---

## 2. 웹의 구성요소

| 구성요소              | 예시                      | 역할                                   |
| ----------------- | ----------------------- | ------------------------------------ |
| **클라이언트(Client)** | 웹 브라우저(Chrome, Edge 등)  | 서버에 요청을 보내고, 응답된 내용을 화면에 표시          |
| **서버(Server)**    | Nginx, Apache, Tomcat 등 | 정적 페이지는 그대로 전달, 동적 페이지는 프로그램 실행 후 응답 |
| **네트워크(Network)** | LAN, WAN, Internet      | 클라이언트와 서버 간 **HTTP/HTTPS 통신 경로**     |



![클라이언트-서버 모델](https://lh3.googleusercontent.com/d/1Ytos16Ll_nOcL2KuZw-c8Yv5QvYfIsdt)

- 웹이 브라우저에서 동작하는 방식

  | 단계               | 설명                        |
  | ---------------- | ------------------------- |
  | 1️⃣ 주소 입력        | 사용자가 브라우저에 URL 입력         |
  | 2️⃣ 요청(Request)  | 브라우저 → 서버로 HTTP/HTTPS 요청 전송     |
  | 3️⃣ 처리(Process)  | 서버가 요청을 분석하고 필요한 데이터를 준비  |
  | 4️⃣ 응답(Response) | 서버 → 브라우저로 HTML/JSON 등 전송 |
  | 5️⃣ 렌더링          | 브라우저가 응답 형식에 따라 화면에 표시



## 3. 웹의 주요 형태

- 사용자의 요청에 따라 화면(내용)이 바뀌는가? ( 정적 / 동적 웹 )

  | 형태    |특징                 |
  | --- |------- |
  | **정적 웹 (Static Web)**          | • 사용자의 요청에 따라 내용이 바뀌지 않음<br>• 예: 회사 소개 페이지                          |
  | **동적 웹 (Dynamic Web)**         | • 사용자의 요청이 있을때마다 결과가 달라짐<br>• 서버 또는 클라이언트에서 데이터를 처리하여 화면을 동적으로 생성<br>• 예: 게시판, 검색 엔진                                 |

- 브라우저에 있는 화면은 어디에서 만들까? ( CSR과 SSR 렌더링 방식 )

  | 단계                | CSR (Client Side Rendering) | SSR (Server Side Rendering) |
  | ----------------- | --------------------------- | --------------------------- |
  | HTML 생성 위치        | 브라우저( JavaScript )            | 서버 ( JSP, PHP, Node.js 등 )    |
  | 화면에 표시(Rendering) | 브라우저                        | 브라우저                        |
  | 첫 화면 속도           | 느릴 수 있음 ( HTML 없이 JS부터 받음 )   | 빠름 ( 완성된 HTML을 바로 받음 )        |
  | SEO(검색 노출)        | 불리함                         | 유리함                         |



  > 현재 대부분의 웹사이트는 CSR과 SSR 방식을 적절히 혼합하여 **동적 웹** 형태로 동작합니다.  
  예) 네이버, 유튜브, 인스타그램 등

## 💡 **요약정리**  
> 1. 웹은 **HTTP/HTTPS를 사용하여 클라이언트와 서버가 정보를 주고받는 시스템**이다.  
> 2. 웹은 ‘내용이 바뀌는가’에 따라 정적 또는 동적으로 나뉘고, 화면(HTML)은 ‘어디에서 만들어지는가’에 따라 CSR 또는 SSR 방식으로 렌더링된다.
