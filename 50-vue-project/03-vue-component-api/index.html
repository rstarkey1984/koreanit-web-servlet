내용:

이번 시간에는 Vue 3의 Composition API를 활용해서
서블릿 기반 게시판 API와 연동하는 SPA를 만들어 봅니다.

1. 더미 데이터로 UI 먼저 구성
2. fetch()로 API 실제 연결
3. 컴포넌트 간 상태/이벤트 흐름 이해

※ 실제 구현 시에는 선택한 게시글 ID를 기준으로 데이터를 가져오고, “수정” 클릭 시 아래 폼 화면으로 전환한다.

게시글 작성 / 수정 화면 (form)

POST /api/board (작성), PUT /api/board/{id} (수정) 에 사용할 폼. UI 구조만 구현해 놓은 상태.

※ “수정 모드”일 때는 기존 제목/내용으로 채워넣고, 저장 시 create vs update를 분기하는 로직을 나중에 추가할 것이다.

SPA 화면 상태 흐름 요약

실제 Vue SPA에서는 currentView 같은 상태로 화면을 전환하게 된다.
  • list → 게시글 목록 화면
  • detail → 게시글 상세 화면
  • form → 게시글 작성/수정 화면

지금은 세 화면을 모두 한 번에 보여주는 “디자인 목업” 상태이고,
나중에 Vue 3 + Composition API로 화면을 쪼개서 컴포넌트로 만들고 v-if 등으로 토글할 예정이다.

이 파일은 수업용 순수 HTML 목업입니다. 이후 단계에서 Vue / API를 한 단계씩 붙일 예정입니다.
